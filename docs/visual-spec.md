# Visual Spec

This document captures the final visual rules for `review-pdf-skill`.

## Overall deck

- Background: warm cream/off-white.
- Typography: clean sans-serif, bold black headings.
- Style: modern landing-page/deck layout.
- Copy: minimal and plain-language.
- Pages: one idea per page.
- Colors:
  - Pink = problem/request.
  - Green = solution/fix.
  - Black = normal text.
  - Blue = links.

## Page 1

Page 1 is always the request + overview page.

Required structure:

1. Top header row.
2. Request eyebrow label.
3. Large document title.
4. Short summary subtitle.
5. Legend pills.
6. Main overview card.

The overview card must include:

- White rounded rectangle.
- Subtle border and soft shadow.
- Hot-pink accent bar near the top-left.
- Heading: `What this document says`.
- 3–5 row cards.
- Pastel numbered circles.
- Clean column alignment.

Do not add any additional blank card/container under this overview card.

## Overview rows

Rows should be subtle white/light-gray rounded cards inside the overview card.

Preferred row labels:

- `Audio link` if audio exists.
- `Original ask`.
- `Found blocker` or `Main issue`.
- `Show fixes` or `Final output`.

If there is no audio, use:

- `Original ask`.
- `What I checked`.
- `Main finding`.
- `Final output`.

## Problem pages

Problem pages use pink.

- Label: `PROBLEM`.
- One clear title.
- One short sentence.
- Screenshot/mockup/card if useful.
- Use arrows/callouts only if they clarify the issue.

## Solution pages

Solution pages use green.

- Label: `SOLUTION`.
- One clear title.
- One short sentence.
- Checklist/mockup/action card.

## Footer

The footer must be minimal.

Use:

- external-link icon + `Voice note`
- vertical divider
- external-link icon + `Resources`

Rules:

- No footer card.
- No footer pill.
- No stacked `VOICE LINK` / `TASKS + RESOURCES` labels.
- Links should be close together and centered.
- Icons and text should share a visual baseline.
- Links must be clickable.

## Link icon

Use an external-link icon, not a chain-link icon.

Shape:

- small blue outlined square/corner.
- diagonal up-right arrow.
- lightweight stroke.

Avoid:

- chunky interlocking circles.
- emoji-style icon.
- oversized icon.
- underline by default.

## QA

Render before delivery:

- Page 1 full render.
- Page 1 overview crop.
- Footer crop.
- Full contact sheet.

Reject output if any of these appear:

- Extra blank container.
- Clipped text.
- Overlap.
- Unsafe margins.
- Footer container.
- Bulky icon.
- Too much text.
- Generic PDF styling.
